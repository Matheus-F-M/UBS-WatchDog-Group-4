UMA NOTA PARA O BACKEND ANTES DE COMEÇAR
    Para haver um parelhamento perfeito entre o backend e o frontend é de suma importância o desenvolvedor front
    (no caso Pat) saber quais parâmetros estão no back e quais são os seus nomes.

    No Frontend, existe uma função que mapeia o nome dos parâmetros do backend para o nome dos parâmetros sendo
    usados no frontend. Por isso, não é extremamente necessário que os nomes dos parâmetros sejam os mesmos.

    Entretanto, é sim importantíssimo que os data types do back e do front estejam alinhados. Caso o contrário,
    o desenvolvedor front precisa ser informado desses "types mismatches" para que seja incluído uma conversão de
    tipos dentro das próprias funções de mapeamento.

    Por isso, convido à tanto os desenvolvedores front quanto os desenvolvedores back desenvolverem uma text file
    como esta para que não haja conflito entre os tipos do frontend e do backent e para que o nome das funções da
    API sejam iguais.

    Boa leitura!
